問1 — ルータの更改
【シナリオ概要】
A社（従業員800人・建設機械販売）は大阪本社と東京支社を広域イーサ網＋IPsec VPN（冗長）で接続。C社SaaSともWAN経由で接続。ルータのメーカーサポート終了に伴い、更改プロジェクトを立ち上げ、担当Bさんが手順書を作成する。
図1 現在のA社NW（構成図）
  インターネット ─── FW10 ─── L3SW10 ─── L2SW11, L2SW12 ─── PC（大阪内部NW）
                          │
                        R10, R11 ─── 広域イーサ網 ─── R30, R31 ─── C社SaaS
                          │
                        R20, R21 ─── IPsec VPN ───(東京支社)
                                         │
                                       L2SW20 ─── PC（東京内部NW）

  ※ OSPFエリアは全体で エリアID 0.0.0.0 の バックボーンエリアのみ
  ※ ルータIDは ループバックIPアドレス を使用
  ※ VRRPはR20(優先度105)、R21(100)でデフォルトゲートウェイを冗長化
  ※ OSPFコスト: 広域イーサ=100, IPsec VPN=1000, ループバック=1, その他=10

設問1 — 現在のA社NW

設問1(1)-a
OSPFのエリアは、エリアIDが0.0.0.0と表記される [ a ] エリアだけで構成されている。
▼
// ANSWER
バックボーン
解説：OSPFにはエリアという概念があり、エリアID=0.0.0.0（または0）が「バックボーンエリア」と呼ばれる特別なエリアです。単一エリア構成ではバックボーンエリアのみ使用されます。エリアを分割するとABR（エリア境界ルータ）が必要になりますが、本問では不使用。

試験ポイント：エリアID=0.0.0.0 ⇒ バックボーンエリアは必ず覚える。
設問1(1)-b
ループバックIPアドレスは、OSPFルータを識別する [ b ] IDとして利用されている。
▼
// ANSWER
ルータ
解説：OSPFではルータを一意に識別するためにルータIDを使用します。通常はループバックインタフェースのIPアドレスがルータIDに採用されます。ループバックは物理的なリンクダウンに左右されないため、安定したIDとなります。
設問1(1)-c
R20, R21の内部NWと接続するインタフェースでは、PCに設定するデフォルトルートのネクストホップに相当する [ c ] の可用性を向上するために、VRRPが動作している。
▼
// ANSWER
デフォルトゲートウェイ
解説：PCのデフォルトルートのネクストホップ＝デフォルトゲートウェイです。VRRPは仮想IPアドレスを使って複数ルータでデフォルトゲートウェイを冗長化するプロトコル。R20（優先度105）が通常マスター、障害時にR21（優先度100）が引き継ぐ設計。
設問1(2)
OSPFのネイバー認証によって、どのような問題を防止できるか。経路交換の安全性の観点から35字以内で答えよ。
▼
// ANSWER
不正なOSPFルータと隣接関係になり経路交換してしまう問題
解説：OSPFネイバー認証がない場合、悪意のあるルータや誤設定された機器がOSPFに参加して偽の経路情報を広告できます。認証（MD5等）によりネイバー確立時に認証を行い、不正機器の参加を防ぎます。

キーワード：「不正なルータ」「隣接関係（ネイバー）」「経路交換」の3要素を含める。
設問1(3)
R31がeBGPで経路広告するAS_PATH長はR30と比較してどうする必要があるか。
▼
// ANSWER
長くする
解説：eBGPのAS_PATHプリペンド設定により、通常はR30経由（AS_PATH短い）、障害時はR31経由（AS_PATH長い）という優先制御を実現します。BGPではAS_PATHが短い経路が優先されるため、R31のAS_PATHを意図的に長くすることで「R30優先、R31バックアップ」の設計になります。

AS_PATHプリペンド：自ASのAS番号を経路広告時に複数回付加する技術。
設問1(4)
プリエンプトディレイを設定し、プライオリティの値が高いルータの電源をオンにした際に一定時間待つのは、ルータがどのような状態になることを待つためか。OSPFを利用していることに着目して25字以内で答えよ。
▼
// ANSWER
OSPFの経路情報が収束した状態
解説：ルータが再起動直後はOSPFの経路交換（ネイバー確立→LSA交換→SPF計算）がまだ完了していません。この状態でVRRPマスターになってしまうと、正常な経路情報なしにトラフィックを受け取り通信断が発生します。プリエンプトディレイでOSPFコンバージェンス完了を待ちます。

収束（コンバージェンス）：全ルータが同一の経路情報を持つ安定状態になること。

設問2(1)

// ANSWER

設問2(2)
本文中の [ d ] に入れる適切な字句を答えよ。（インターネット接続サービスの種別変更に関する空欄）
▼

解説：ベストエフォート型はネットワークの混雑状況によって実効速度が変動します。図2から実効速度が約200Mbpsで頭打ちになっており、帯域が制限されていることが判明。解決策として「帯域確保（ギャランティ）型」への変更を提案。帯域確保型はSLAで最低帯域が保証されます。
設問2(3)
NAPTエントリーを作成する際に、ルータ自身が割り当てている値を、(i)〜(v)の記号で全て答えよ。
▼
// ANSWER
(iii) registered IP address、(iv) assigned port number
解説：NAPTエントリーの各フィールド：
(i) local IP address：PCのプライベートIPアドレス（PCが持つ値）
(ii) local port number：PCが使用するポート番号（PCが決める）
(iii) registered IP address：ISPから割り当てられたグローバルIP → ルータが保持・割り当て
(iv) assigned port number：NAPTが変換後に使うポート番号 → ルータが割り当て
(v) IP protocol：TCP/UDP（PCのパケットから取得）

ルータはグローバルIPとポート番号（変換後）を管理・割り当てます。
設問2(4)
PC1台あたりのセッション数を最大200と見積もり、全PCが問題なく利用できるPC台数の上限と、必要なNAT変換テーブルのエントリー数を整数で答えよ。

設問3 — HTTPバージョンとNAPTとの関係

HTTPバージョンとNAT変換エントリー数の関係
  HTTP/1.1 → TCPコネクション×複数 → NAPTエントリー×複数（HoLブロッキング回避のため）
  HTTP/2   → TCPコネクション×1   → NAPTエントリー×1（ストリームで多重化）
  HTTP/3   → QUICコネクション×1  → NAPTエントリー×1（UDPベース、TCPのHoL回避）
             (UDP使用)
設問3(1)
HoL（Head of Line）ブロッキングについて、全ての処理が完了するまでに時間が掛かる理由を30字以内で答えよ。
▼
// ANSWER
先行の処理が終わるまで後続の処理が待たされるから
解説：HoL（Head of Line）ブロッキングは、列の先頭（Head）にある処理（Line）が詰まると、後続の処理が全て待機させられる問題です。HTTP/1.1ではリクエストを逐次処理するため、1つの画像ダウンロードが遅いと後続のリクエストが全て待たされます。
設問3(2)
本文中の [ e ]〜[ h ] に入れる適切な字句を答えよ。
▼
// ANSWER
e: リクエスト（PCからサーバへのHTTPリクエスト）
f: レスポンス（サーバからPCへのHTTPレスポンス）
g: ストリームID（HTTP/2の多重化識別子）
h: 並列（HTTP/2で複数ストリームを同時処理）
解説：HTTP/1.1で10枚の画像をダウンロードする場合、HTTPリクエストとレスポンスを1つずつ10回実行。HTTP/2ではストリームID（整数）を付与して1つのTCPコネクション内で並列処理できます。
設問3(3)
EサービスのサーバがHTTP/3に対応している場合、NATエントリーを有効活用するための保持時間の調整を答えよ。また、その効果を45字以内で答えよ。
▼
// ANSWER
調整：UDPの保持時間を300秒より小さくする
効果：未使用のUDPフローによるNAPT変換エントリーを早期に削除できる
解説：HTTP/3はQUIC（UDP）を使用します。UDPのNAT保持時間はデフォルト300秒と長めです。HTTP/3のQUICコネクションが終了しても、UDPエントリーが300秒間保持されると、その分テーブルが無駄に消費されます。保持時間を短くすることで不要なエントリーを早期解放し、テーブルを有効活用できます。

ただし短すぎると通信中のフローが切断されるため注意が必要（本文でも「極端な値は別の問題が発生」と記述）。

b: 認証サーバ（プロキシサーバからUDP/1812でRADIUS認証）

• 項番1：DMZ→[a]、UDP/514 → syslogプロトコル（UDP514）。DMZのNTPサーバ・プロキシサーバのログをsyslogサーバに転送。
• 項番3：プロキシサーバ→[b]、UDP/1812 → RADIUSプロトコル（UDP1812）。プロキシが利用者認証のためにRADIUSサーバ（認証サーバ）に問い合わせ。
設問1(5)
表1のFW許可ルール中の [ c ] [ d ] に入れる適切な数字を答えよ。
▼
// ANSWER
c: 10080（内部セグメント→プロキシサーバへのTCP接続ポート）
d: 443（内部セグメント→K社SaaS、HTTPS直接接続）
解説：
• c：本文に「プロキシサーバはTCP/10080で接続を待ち受ける設定」と明記。PCからプロキシへはTCP/10080。
• d：K社SaaSへのアクセスはHTTPS（TCP/443）。PACファイルで業務用サーバとK社SaaSへはプロキシを経由せず直接接続する設定のため、FWにTCP/443の直接許可ルールが必要。

設問2 — SWGサービスの仕様調査

設問2(1)
SWGサービスがHTTPS通信のコンテンツを調査するために、あらかじめPCにインストールする証明書を20字以内で答えよ。
▼
// ANSWER
SWGサービスのルート証明書
解説：SWGサービスはHTTPS通信をSSL/TLSインスペクション（中間者攻撃的な手法）で検査します。SWGがHTTPS通信を復号・検査・再暗号化する際、SWG自身が発行した証明書をPCに提示します。PCがこれを信頼するためには、SWGのルート証明書（CA証明書）をあらかじめOSやブラウザの信頼済み証明書ストアにインストールする必要があります。
設問2(2)
マルウェア駆除機能の隔離環境上での動作時に、ファイルが遠隔操作可能なマルウェアを含む場合に観察される外部通信先の名称を答えよ。
▼
// ANSWER
C&Cサーバ
解説：C&C（Command and Control）サーバは、マルウェアに感染した端末を遠隔操作するための指令サーバです。マルウェアは感染後、C&Cサーバに接続して命令を受け取ります。サンドボックス（隔離環境）でマルウェアを実行すると、C&Cサーバへの通信試行が観察されます。これにより遠隔操作型マルウェア（RAT等）を検知できます。

設問3(1)

設問3(2)
(B)R-PCから(K)業務用サーバに接続する場合のアクセス経路を、図2中の(A)〜(M)を用いて答えよ。

// ANSWER
(B) → (A) → (H) → (I) → (E) → (I) → (K)

(B)R-PC → (A)ソフトE → インターネット → (H)SWGサービス → (I)FW → (E)リバースプロキシサーバ → (I)FW → (K)業務用サーバ

R-PCからの通信はソフトEを経由してSWGサービスを通ります。業務用サーバへはリバースプロキシサーバ経由でアクセスします。

設問3(4)

// ANSWER

設問3(5)
業務用サーバのプライベートIPアドレスを直接接続リストに登録することで得られる利点を、通信の経路に着目して10字以内で答えよ。

// ANSWER
FWの負荷軽減（または「RTTの短縮」「経路の短縮」）

解答の要点は「直接通信・経路短縮による利点」であること。
